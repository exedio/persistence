COPE

Cope with Object Persistence

Overview

COPE provides a persistence layer for Java. It allows you to specify the persistent schema with simple constructs within your java source code. No XML files need to be written. Everything else, such as setting up database tables, choosing datatypes for persistent fields, loading and writing objects etc is done for you. COPE does also provide a search facility, which allows to write complex queries using java objects, providing type safety at compile-time.
COPE is open source software and available under the GNU Lesser General Public License. For getting started with COPE and as a template for your own projects there is a simple demo application available under the The MIT License.

Documentation

Documentation is organized into trails, which show you how to use cope on step-by-step examples.
  • Introduction Trail tells you how to get started with cope in your project.
  • Attribute Trail tells you what kind of data can be stored using cope.
  • Transactions Trail tells you how to use transactions in cope.
  • Searching Trail gives you an introduction into the searching capabilities of cope.
  • Attribute Reloaded Trail covers all the more specific possibilities to store data with cope not mentioned in the Attribute Trail.
  • Web Application Trail shows you the little differences when using cope within a web container.
  • API Documentation

Downloads

Downloads are hosted on sourceforge.
The development process of COPE is strongly influenced by the ideas of Extreme Programming. It heavily relies on Automated Continuous Integration.
Therefore, releases published on sourceforge do not carry any explicit version number. Instead each release is named after the build number assigned by the cruisecontrol server that built this release. Each build is subject to extensive automated unit testing on all supported databases (currently Hsqldb, Mysql 4+5 and Oracle 9).
